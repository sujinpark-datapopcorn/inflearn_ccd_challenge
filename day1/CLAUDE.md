# B2G 미세먼지 분석 프로젝트

> 이 파일은 작업 폴더 루트(`inflearn_ccd_challenge/`)에 둔다.
> Claude Code가 매번 이 파일을 읽고 같은 전제로 분석한다.
> **이 파일을 그대로 쓰면 강사 시연과 같은 결과가 재현된다.**

---

## 데이터
- 위치
  - `./기간별_일평균_대기환경_정보_2024년.csv`
  - `./기간별_일평균_대기환경_정보_2025년.csv`
- 출처: 서울 열린데이터광장 OA-2218 (일별 평균 대기환경 정보)
- 인코딩: **CP949** (EUC-KR 호환)
- 시간 범위: 2024-01-01 ~ 2025-12-31 (731일 × 25개 자치구)
- 측정소: 서울 25개 자치구
- 컬럼명 통일: 2024 "미세먼지/초미세먼지" → 2025 "미세먼지농도/초미세먼지농도" 와 일치하도록 분석 시 통일 (분석에서는 `pm10`, `pm25`로 사용)

## 전처리 규칙
- `측정일시`는 항상 `datetime`으로 파싱 (`pd.to_datetime`, `format="%Y%m%d"`)
- 분석에는 `pm10`, `pm25`만 사용 (다른 가스는 별도 요청 시)
- 결측은 `dropna(subset=['pm25'])` 또는 `dropna(subset=['pm10'])` 로 처리
- 이상치 기준: **IQR 3배** (황사 spike를 정상값으로 인정)
- 황사 시즌 정의: **3~5월**
- 자치구 그루핑 키: `측정소명` (강북구·종로구 등)

## 차트·시각화 규칙
- 한글 폰트 (Mac: AppleGothic / Windows: Malgun Gothic)
- seaborn은 `sns.set_theme()` **먼저 호출** 후 한글 폰트 재지정
- 시계열은 라인, 카테고리 비교는 막대/박스플롯
- 자치구 막대 정렬: **PM2.5 내림차순** (가장 나쁜 자치구가 왼쪽)
- 연도별 비교 색상 고정: 2024 = `#94A3B8` (회색), 2025 = `#4A7BF7` (파랑)
- 황사 시즌 음영: `axvspan(2.5, 5.5, color="#FBBF24", alpha=0.15)`
- y축 단위 명시 (㎍/㎥)

## 출력
- 차트: PNG로 `./outputs/` 폴더에 저장 (`mkdir -p outputs`)
- 파일명 규칙
  - 자치구 비교: `yoy_district_pm25.png`
  - 월별 추이: `yoy_monthly_pm25.png`
  - 그 외 분석은 `analysis_<주제>.png` 패턴
- 결과 텍스트: 한글, 코드 설명은 한 줄, 인사이트는 1~2 문장
- 시간 범위 답할 때 항상 연도 명시 ("2024년", "2025년", "두 해 통합")

## 재현·재사용·반복
- 위 규칙을 그대로 두면 강사 시연과 같은 결과가 나온다

- 버전 구분(v1/v2) 안 함 — 폴더(`day1/`, `day2/`)로 히스토리 관리
