## 시연 — 통합 EDA

```
seoul_air_quality_2024.csv + seoul_air_quality_2025.csv 통합해서 EDA 해줘.
순서: shape → dtypes → 결측 비율 (연도별) →
자치구별 PM2.5 연평균 비교 (2024 vs 2025) →
월별·요일별 시즌성 → 이상치 후보 (연도별 spike Top 5) → 등급 분포 변화.
차트 다 그려서 ./outputs/ 에 저장하고, 핵심 인사이트 5줄로 요약.
```

## 4단계 — 단독 → 결합 흐름

### 1단계 — 단독 분석
```
"내가 사는 자치구의 2025 PM2.5 평균은?" 같은 식으로
본인이 고른 질문 1개를 그대로 던져봐.
결과는 표 + 한 줄 인사이트로.
```

### 2단계 — 따릉이 + 지하철 결합
```
미세먼지 등급(좋음/보통/나쁨/매우나쁨)별 따릉이 일평균 대여 건수 보여줘.
같은 분석을 지하철 일별 총 승하차로도 한 번 더.
두 결과를 한 화면에 나란히 그려서
"환경에 민감한 교통수단" 비교 메시지가 나오게.
```

### 3단계 — 날씨 결합
```
weather_seoul_daily.csv 도 같이 읽어서,
PM2.5와 풍속·강수·기온의 상관계수를 표로 만들고,
풍속이 셀수록 PM2.5가 떨어지는지 산점도로 검증해줘.
황사 시즌·비황사 시즌 따로 봐줘.
```

### 4단계 — 언급량 결합
```
buzz_keywords.csv 의 "미세먼지" 검색량과 실제 PM2.5 일별 시계열을
같이 그려줘. 검색량이 PM 농도를 며칠 따라가는지(상관 lag) 알려주고,
사람들이 PM 몇 ㎍/㎥부터 검색하기 시작하는지 임계값 추정해줘.
```

## CLAUDE.md 자연어 누적

```
오늘 EDA에서 발견한 규칙을 CLAUDE.md에 추가해줘.
- 측정일시는 항상 datetime으로 파싱
- 결측 50% 이상 측정소는 분석에서 제외
- 이상치 기준은 IQR 3배 (황사 spike는 정상값으로 인정)
- 황사 시즌 = 3~5월
- 결합 데이터는 './data/' 하위, 날짜 컬럼은 'date'로 통일
- 시즌별 비교 시 4분위 컬러 팔레트 통일
- y축 단위 항상 명시 (PM은 ㎍/㎥, 따릉이는 건, 지하철은 명)
기존 규칙과 중복은 합치고, 어울리는 섹션 위치에 추가해줘.
```
