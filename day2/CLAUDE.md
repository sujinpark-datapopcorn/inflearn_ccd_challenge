# 서울 미세먼지 분석 프로젝트

> Day 1 CLAUDE.md를 그대로 복사한 다음, Day 2 EDA에서 발견한 규칙을 자연어로 누적한다.
> "오늘 발견한 규칙을 CLAUDE.md에 추가해줘" 한 마디면 Claude Code가 알아서 끼워넣는다.

## 데이터
- 위치: 작업 폴더 하위 — `data/seoul_air_quality/`, `data/seoul_bike/`, `data/seoul_subway/`, `data/weather/`, `data/buzz/`
- 인코딩: CP949 — 미세먼지·따릉이·지하철·날씨 CSV / UTF-8 — buzz CSV
- 출처:
  - 미세먼지: 서울 열린데이터광장 OA-2218 (일별 평균 대기환경)
  - 따릉이: 서울 열린데이터광장 (일별 대여 건수)
  - 지하철: 공공데이터포털 (서울교통공사 역별 일별 시간대별 승하차)
  - 날씨: 기상청 ASOS 서울(108) 일자료
  - 언급량: 네이버 데이터랩 + 구글 트렌드
- 시간 범위: 2024-01-01 ~ 2025-12-31
- 측정소: 서울 25개 자치구
- 컬럼명 통일: 2024 "미세먼지/초미세먼지" → 2025 "미세먼지농도/초미세먼지농도" 와 일치
- **결합 데이터의 날짜 컬럼은 항상 `date`로 통일**

## 분석 규칙
- 모든 결과는 한글로 출력
- 차트 한글 폰트 깨지지 않게 처리 (Mac: AppleGothic, Windows: Malgun Gothic)
- seaborn 사용 시: `sns.set_theme()` 먼저 호출 → 그 다음 폰트 재지정
- 차트는 PNG로 `./outputs/` 폴더에 저장
- 측정일시는 `datetime`으로 파싱
- 시간 범위 답할 때 항상 연도 명시
- **결측 50% 이상 측정소는 분석에서 제외**
- **이상치 기준은 IQR 3배** (황사 spike는 정상값으로 인정)
- **황사 시즌 정의: 3~5월**
- 파생 컬럼(`month`, `weekday`, `season_type`)은 EDA 내부 계산용 — 결과 출력 컬럼으로 노출 X

## 시각화 규칙
- 시계열은 라인, 카테고리 비교는 박스플롯
- 시즌별 비교 시 4분위 컬러 팔레트 통일 (봄·여름·가을·겨울)
- y축 단위 항상 명시 (PM은 ㎍/㎥, 따릉이는 건, 지하철은 명)
- 결합 차트는 한 화면에 두 데이터를 같이 — 단순 나란히 두기 X

## 출력
- 코드 설명은 한 줄 요약
- 결과 인사이트는 핵심 1~2 문장
